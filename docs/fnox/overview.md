# fnox — что это и зачем

## Проблема

Секреты — пароли от БД, API-ключи, JWT-токены — нельзя хранить в репозитории открытым текстом. Обычно их кладут в `.env` файл и добавляют в `.gitignore`. Это работает, но создаёт другую проблему: у каждого разработчика свой набор переменных, который передаётся через мессенджеры, почту или вообще теряется.

fnox решает это по-другому — секреты хранятся **прямо в репозитории**, но **зашифрованными**.

## Как это работает

fnox использует [age](https://age-encryption.org/) — современный инструмент шифрования. Каждый разработчик генерирует пару ключей: публичный и приватный.

- **Публичный ключ** — указывается в `fnox.toml` в списке `recipients`. Он шифрует данные и не является секретом.
- **Приватный ключ** — лежит локально на машине разработчика. Он расшифровывает данные и никуда не коммитится.

```
fnox.toml (в репо) ──── зашифровано публичным ключом ────┐
                                                           │
~/.config/age/key.txt ── расшифровывается приватным ключом ┘
```

Клонировал репо — запустил `mise install` — fnox автоматически расшифровывает нужные переменные используя твой приватный ключ.

## Что хранится в репозитории

```toml
# fnox.toml
[providers.main]
type = "age"
recipients = ["age1ldm5xzq85pu7..."]  # публичный ключ — не секрет

[secrets]
DATABASE_URL = { provider = "main", value = "YWdlLWVuY3J5cHRpb24..." }
#                                             ^^^^^^^^^^^^^^^^^^^^
#                              зашифрованное значение — без приватного ключа бесполезно
```
