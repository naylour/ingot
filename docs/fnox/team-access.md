# Управление доступом в команде

## Новый участник: генерация ключа

Каждый разработчик генерирует свою пару ключей один раз. Приватный ключ никуда не передаётся и не коммитится.

```bash
mkdir -p ~/.config/age
age-keygen -o ~/.config/age/ingot-secret.txt
```

Команда создаст файл с приватным ключом и выведет публичный:

```
Public key: age1abc123def456...
```

Публичный ключ нужно передать тому, кто управляет доступом.

## Выдать доступ

1. Добавь публичный ключ нового участника в нужный провайдер в `fnox.toml`:

```toml
[providers.backend]
type = "age"
recipients = [
    "age1существующий...",
    "age1новый...",          # ← добавили
]
```

2. Перешифруй секреты с учётом нового ключа:

```bash
fnox reencrypt
```

`reencrypt` расшифровывает все текущие значения твоим приватным ключом и шифрует заново — уже под весь обновлённый список recipients. После этого новый участник сможет читать секреты.

3. Закоммить изменения в `fnox.toml` и смержи через PR.

## Отозвать доступ

1. Удали публичный ключ участника из `recipients`.
2. Запусти `fnox reencrypt`.
3. Закоммить и смержи.

После этого старые зашифрованные значения, которые могли остаться у участника локально, стали бесполезны — данные перешифрованы без его ключа.

## Защита на уровне репозитория

`fnox reencrypt` требует приватный ключ существующего участника — значит, самостоятельно выдать себе доступ нельзя. Но участник с доступом технически может добавить кого угодно в `recipients`.

Чтобы это контролировать — настрой CODEOWNERS на GitHub:

**`.github/CODEOWNERS`:**
```
fnox.toml @твой-github-username
```

Тогда любой PR, который меняет `fnox.toml`, потребует твоего approve. Без него не смержится.

Не забудь включить в настройках репозитория:
**Settings → Branches → Add ruleset → Require review from Code Owners**

## Итог: кто что может

| Действие | Требует |
|---|---|
| Читать зашифрованные значения | свой приватный ключ в recipients |
| Добавить нового участника | приватный ключ + `fnox reencrypt` |
| Изменить `fnox.toml` в main | approve от CODEOWNERS |
| Отозвать доступ | приватный ключ + `fnox reencrypt` |
