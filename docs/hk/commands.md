# Команды hk

## Основные команды

### hk check

Запускает проверки в режиме **только чтения** — ничего не изменяет, только сообщает об ошибках.

```bash
hk check                    # staged файлы
hk check --all              # все файлы в проекте
hk check src/lib/utils.ts   # конкретный файл
```

Удобно запускать в CI — там не нужно ничего исправлять, только знать есть ли ошибки.

### hk fix

Запускает форматтеры и линтеры в режиме **автоисправления**.

```bash
hk fix                      # staged файлы
hk fix --all                # все файлы в проекте
hk fix src/                 # файлы в папке
```

### hk run

Запускает конкретный хук вручную.

```bash
hk run pre-commit           # как будто делаешь git commit
hk run pre-push             # как будто делаешь git push
hk run pre-commit --check   # в режиме только проверки
```

## Отладка и диагностика

### Посмотреть план без запуска

```bash
hk run pre-commit --plan
```

Покажет какие шаги будут запущены, на каких файлах и почему:

```
pre-commit
  biome [fix] → src/lib/utils.ts, src/routes/+page.svelte
```

### Понять почему файл включён или исключён

```bash
hk run pre-commit --why biome
```

Объяснит по каждому файлу — подходит ли он под glob шага и почему.

### Запустить только один шаг

```bash
hk run pre-commit --step biome
```

Полезно когда шагов несколько и хочешь проверить только один.

### Пропустить шаг

```bash
hk run pre-commit --skip-step conventional
```

## Проверка изменений в PR

Одна из самых полезных возможностей — запустить проверку только на файлах которые изменились в текущей ветке относительно main:

```bash
hk check --pr
```

Это эквивалентно:

```bash
hk check --from-ref origin/main --to-ref HEAD
```

Удобно перед тем как открыть PR — быстро проверить только свои изменения, не весь проект.

## Полностью пропустить хук

В крайнем случае когда нужно закоммитить несмотря на ошибки:

```bash
git commit --no-verify
```

Использовать только если понимаешь что делаешь — хуки существуют не просто так.
