# hk — что это и зачем

## Проблема

Git hooks существуют давно — можно написать скрипт в `.git/hooks/pre-commit` и он будет запускаться перед каждым коммитом. Но у этого подхода есть проблемы:

- `.git/` не коммитится — у каждого в команде хуки настроены по-своему или не настроены вообще
- Скрипты медленные — запускают всё подряд, даже если изменился один файл
- Нет параллельности — всё выполняется последовательно

hk решает это: хуки описываются в `hk.pkl` в репозитории, запускаются параллельно и только на изменённых файлах.

## Что умеет hk

- Запускает линтеры и форматтеры **только на staged файлах** — не тратит время на весь проект
- **Параллельно** — несколько шагов идут одновременно
- **Автоматически исправляет** и стейджит изменения — сам делает `git add` после форматирования
- Поддерживает **профили** — одни проверки при коммите, другие при пуше, третьи только в CI
- Умеет проверять только **файлы из текущего PR** — удобно перед пушем

## Формат конфига

hk использует [PKL](https://pkl-lang.org/) — язык конфигурации от Apple. Файл называется `hk.pkl`.

PKL даёт типизацию и автодополнение в редакторе (нужен PKL language server). Для **запуска** hk Java не нужна — в проекте настроен встроенный evaluator `pklr` через `HK_PKL_BACKEND=pklr` в `mise.toml`.

Структура конфига:

```pkl
amends "package://github.com/jdx/hk/releases/download/v1.46.0/hk@1.46.0#/Config.pkl"
import "package://github.com/jdx/hk/releases/download/v1.46.0/hk@1.46.0#/Builtins.pkl"

hooks {
    ["pre-commit"] {         // хук git
        fix   = true         // запускать в режиме исправления
        stash = "git"        // прятать unstaged изменения перед запуском

        steps {
            ["biome"] {      // шаг внутри хука
                glob  = List("**/*.ts", "**/*.svelte")
                check = "biome check {{files}}"
                fix   = "biome check --write {{files}}"
            }
        }
    }
}
```

## Как подключить к проекту

После клонирования — один раз:

```bash
hk install
```

Это создаёт скрипты в `.git/hooks/`, которые вызывают hk. Каждый новый участник делает то же самое.
