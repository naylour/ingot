# Переменные окружения в mise

## Блок [env]

Кроме инструментов, mise умеет устанавливать переменные окружения для проекта. Когда заходишь в папку — переменные появляются. Когда выходишь — пропадают.

```toml
[env]
APP_ENV    = "development"
LOG_LEVEL  = "debug"
```

Это удобно для переменных, которые не являются секретами — названия окружений, флаги, пути.

## Шаблоны

В значениях можно использовать шаблоны `{{env.НАЗВАНИЕ}}` для обращения к системным переменным окружения:

```toml
[env]
FNOX_AGE_KEY_FILE = "{{env.HOME}}/.config/age/ingot-secret.txt"
```

`{{env.HOME}}` раскроется в `/home/твоё-имя`. Без шаблона тильда `~` не раскрывается — mise передаёт её буквально как строку.

## Связь с fnox

В этом проекте `[env]` используется именно для того, чтобы сказать fnox где лежит приватный ключ для расшифровки секретов:

```toml
# mise.toml
[tools]
age  = "1.3.1"
fnox = "1.25.1"

[env]
FNOX_AGE_KEY_FILE = "{{env.HOME}}/.config/age/ingot-secret.txt"
```

Каждый разработчик генерирует свой ключ и кладёт его по этому пути. mise автоматически подхватывает переменную при входе в проект — fnox знает где искать ключ.

Подробнее о самом fnox — [здесь](../fnox/overview.md).
