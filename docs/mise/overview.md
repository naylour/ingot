# mise — что это и зачем

## Проблема

У каждого в команде на машине стоят разные версии Node, Bun, инструментов. У одного `bun@1.0`, у другого `bun@1.2`. Проект ведёт себя по-разному — и непонятно почему.

Решение — зафиксировать версии инструментов прямо в репозитории.

## Что такое mise

mise — менеджер версий инструментов. Аналог nvm, но умеет работать с чем угодно: Bun, Node, Go, Python, Rust-тулчейном, и вообще любым инструментом из реестра.

Конфигурация лежит в `mise.toml` рядом с кодом. Когда ты заходишь в папку проекта — mise автоматически переключается на нужные версии.

```toml
# mise.toml
[tools]
bun = "1.2.0"
node = "22.0.0"
```

Зашёл в папку — у тебя `bun@1.2.0`. Вышел — вернулся к системному. Никаких ручных переключений.

## Чем отличается от nvm

|                      | nvm / nvmrc | mise                  |
| -------------------- | ----------- | --------------------- |
| Языки                | только Node | любой инструмент      |
| Конфиг               | `.nvmrc`    | `mise.toml`           |
| Переменные окружения | нет         | есть (`[env]` блок)   |
| Автопереключение     | нет         | да (через shell hook) |
