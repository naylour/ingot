# Доступ к секретам

Секреты в проекте зашифрованы через age. Чтобы их читать — нужен личный приватный ключ, и твой публичный ключ должен быть добавлен в список допущенных.

## 1. Сгенерируй ключ

```bash
mkdir -p ~/.config/age
age-keygen -o ~/.config/age/ingot-secret.txt
```

Команда создаст файл с приватным ключом и выведет публичный в терминал:

```
Public key: age1abc123def456...
```

**Приватный ключ никуда не передаётся и не коммитится.** Он лежит только у тебя локально.

## 2. Передай публичный ключ

Скопируй публичный ключ (`age1abc123...`) и отправь тому, кто управляет доступом в проекте. Он добавит твой ключ в `fnox.toml` и перешифрует секреты.

Посмотреть свой публичный ключ в любой момент:

```bash
grep "public key" ~/.config/age/ingot-secret.txt
```

## 3. Проверь что всё работает

После того как тебя добавили и изменения попали в `main` — подтяни репозиторий и проверь:

```bash
git pull
fnox list
```

Если доступ выдан правильно — увидишь список секретов. Попробуй получить значение:

```bash
fnox get MY_SECRET
```

## Если что-то не работает

Убедись что переменная `FNOX_AGE_KEY_FILE` указывает на правильный путь. Mise должен её подставлять автоматически при входе в папку проекта:

```bash
echo $FNOX_AGE_KEY_FILE
# /home/твоё-имя/.config/age/ingot-secret.txt
```

Если переменная пустая — проверь что `mise activate` добавлен в конфиг shell и ты перезапустил терминал после `mise install`.

Подробнее про то как это устроено — [в разделе fnox](../fnox/overview.md).
